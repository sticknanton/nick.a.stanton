JeopPrep Game


  • JeopPrep Link
  • JeopPrep GitHub

JeopPrep is a group project and my first step into the world of Node. It is a single-player Jeopardy game built around the idea of being a player interested in one day competing on the show. All of the data for the questions is from a third-party API and adds up to 217,000 questions from actual jeopardy episodes. To inspire competition we included stat tracking and a leaderboard. In lieu of having the time to perfect checking correct answers (string comparisons) we added a challenge feature. The game itself was coded by me in the limited time we had and all of the backend models, routing and authentication was accomplished by my partner Graig, here's his linked in.

CardSmith and MEAN Blog


  • Meanblog Link | Meanblog GitHub
  • Cardsmith Link | Cardsmith GitHub

These are two small projects just to get myself experienced working within different stacks. The first is a simple MEAN-stack app using a NoSQL database. A blog with user authentication and the ability to post new entries and serach through pre-existing ones. Cardsmith is an app structured by angular with a Rails API. A new user submits a form with a picture URL, other fields and, most importantly, their secret power. A card is generated that resembles a card from a trading card game like Magic: The Gathering and is added to the data and the page.

Restaurant POS Project

  • Restaurant Link
  • Restaurant GitHub

My only large Ruby on Rails project. This is a full-stack restaurant POS sytem built on a psql database. It features authentication for an admin, party management, order placing and separate views for a kitchen and bar screen. There is a separate view for checks and the ability to "close" checks and/or split them.
